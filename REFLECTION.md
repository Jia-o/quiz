1. How far did the agent get
- The agent got really far implementing my spec. It followed my spec so exactly that when I made mistakes or left important steps out it also made those same mistakes and had those same gaps. 4/28 items from the review failed while a few others got warnings. 

2. Where did I intervene 
- One way I had to intervene for the content was to fill in logical gaps in the procedure. For example, in my spec I asked it to ask for login upon opening the app but never told it to create a way to make a new account. Therefore upon first run, I could not create a new account until after I tried and failed a login. 
- Another way I had to intervene was to hold it back from doing too much. When I asked it to do the review it just fixed everything even though I specified to make a text output. 

3. How useful was the AI review
- As mentioned before, the review just fixed everything and then gave itself a perfect review. I had to tell it to go back, read the commit history, and make the markdown file based on the status of the initial code. 
- As for the secondary review, it hit on some really good points that I was thinking too such as “hort-answer grading requires exact wording”. It also pointed out some important security stuff that I don’t understand. 

4. Spec quality → output quality
- I definitely could have improved the spec or at least gave it explicit permission to add additional features that it would benefit from.
- Another way I would improve the spec is by giving it more deliberate problems I like instead of copying from the homework 8 writeup. Most of the issues I had came from disliking the ways the problems were created, presented, and graded. 

5. When would I use this workflow
- I don't like this workflow at all tbh it scares me quite a bit. I would maybe use it for extremely easy or repetitive stuff that, if done by hand, would require a lot of copy-pasting.
